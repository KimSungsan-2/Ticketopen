# NOL 티켓 오픈 스크래퍼 설계

## 목적

인터파크(NOL 티켓)에서 뮤지컬/서울 지역의 티켓 오픈 예정 목록을 매일 자동으로 수집하고, podor.co.kr에 미등록된 항목을 메일로 알려주는 시스템.

## 전체 흐름

```
[GitHub Actions - 매일 오전 9시 KST (cron: 0 0 * * * UTC)]
        │
        ▼
[1. 인터파크 스크래핑 (Playwright)]
   - tickets.interpark.com/contents/notice 접속
   - 장르: 뮤지컬, 지역: 서울 필터 적용 (Playwright로 UI 클릭)
   - 목록 페이지 스크롤/페이지네이션으로 전체 항목 수집
   - 각 공연 상세 페이지 URL 수집
   - 각 상세 페이지에서 추출:
     · 공연명 (원문)
     · 티켓오픈 일시 (날짜 + 시간)
     · 오픈차수 (1차, 2차, 마지막 등)
     · 오픈공연기간 (이번 오픈에 해당하는 기간)
     · 공연장
     · 전체공연기간 (공연 전체 기간, 시즌 매칭에 사용)
   - 어린이 뮤지컬 필터링 (키워드 기반)
   - 페이지 로드 간 1~2초 딜레이 (rate limiting 방지)
   - 각 페이지 타임아웃: 30초, 실패 시 최대 2회 재시도
        │
        ▼
[2. podor API 매칭]
   - POST /accounts/login/ → JWT 토큰 획득
   - GET /api/performance/ → 전체 공연 목록 1회 조회, 메모리에 캐싱
   - GET /api/performance-place/ → 전체 공연장 목록 1회 조회, 메모리에 캐싱
   - GET /api/performance-season/ → 전체 시즌 목록 1회 조회, 메모리에 캐싱
   - 표준 공연명 매칭 → 시즌 매칭 → 공연장 매칭 순서로 처리
        │
        ▼
[3. 중복 체크]
   - GET /api/performance-open/ → 기존 데이터 전체 조회
   - 시즌 id + 기타(오픈차수) 조합으로 중복 확인
        │
        ▼
[4. 결과 분류 & 메일 발송]
   - 신규/미등록/매칭실패/파싱실패/스킵/어린이제외 분류
   - 메일 발송 (수신: tingsung93@naver.com)
   - 메일 발송 실패 시 stdout으로 리포트 출력 (GitHub Actions 로그에 남김)
   - 스크래핑 결과 0건이면 "수집 결과 없음 (스크래핑 오류 가능성)" 경고 메일 발송
```

## 데이터 매핑

인터파크 상세 페이지에서 추출한 데이터를 podor API의 `performance-open` 필드에 매핑:

| podor 필드 | 소스 | 예시 |
|---|---|---|
| `시즌` | season id (전체공연기간으로 시즌 매칭) | `1705` |
| `공연명` | 표준 공연명 (GET /api/performance/ 에서 매칭) | `"은밀하게 위대하게"` |
| `오픈날짜` | 상세 페이지 티켓오픈 날짜 (YYYY-MM-DD) | `"2026-03-17"` |
| `오픈시간` | 상세 페이지 티켓오픈 시간 (HH:MM) | `"11:00"` |
| `기타` | 오픈차수 | `"마지막"` |
| `오픈회차` | 오픈공연기간 | `"4월 11일 - 4월 26일"` |

**두 가지 "기간"의 구분:**
- `전체공연기간`: 공연 전체 기간 (예: 2026.01.30 ~ 2026.04.26). 시즌 매칭에 사용.
- `오픈공연기간`: 이번 티켓 오픈에서 판매되는 회차 기간 (예: 4월 11일 ~ 4월 26일). `오픈회차` 필드에 매핑.

## 어린이 뮤지컬 필터링

공연명에 아래 키워드가 포함되면 제외 (대소문자 무시):
- 어린이, 키즈, Kids, 가족, 패밀리, Family, 주니어, Jr

## 공연명 매칭 로직

1. 인터파크 공연명에서 부제 제거: 콜론(:), 하이픈(-), 괄호 이후 텍스트를 순차적으로 제거하며 매칭 시도
   - 예: "은밀하게 위대하게:THE LAST-10주년 기념공연" → "은밀하게 위대하게:THE LAST" → "은밀하게 위대하게"
2. 공백/특수문자 정규화 후 podor 공연명 목록에서 정확히 일치하는 항목 탐색
3. 정확 일치 실패 시 → podor 공연명이 인터파크 공연명에 포함되거나, 그 반대인 경우 매칭 (substring 매칭)
4. 그래도 실패 → "매칭 실패"로 분류

## 시즌 매칭 로직

1. 표준 공연명으로 해당 공연의 모든 시즌 조회
2. 각 시즌의 `공연기간` (예: "2026.01.30;2026.04.26")을 파싱
3. 인터파크에서 가져온 전체공연기간의 시작일이 시즌 공연기간 범위 내에 포함되는 시즌 선택
4. 여러 시즌이 매칭되면 → 공연기간이 가장 정확히 겹치는 시즌 선택
5. 매칭 실패 → "시즌 미등록"으로 분류

## 중복 체크 기준

`시즌 id` + `기타`(오픈차수) 조합이 기존 performance-open에 존재하면 중복으로 판단하여 스킵. 시즌 id를 포함함으로써 같은 공연의 다른 시즌에서 동일 차수(예: "1차")가 있어도 구분 가능.

## 엣지 케이스 처리

1. **표준 공연명 매칭 실패**: 위 매칭 로직 4단계 적용 후에도 실패하면 "매칭 실패"로 분류.
2. **시즌 매칭 실패**: "시즌 미등록"으로 분류.
3. **podor 미등록 공연**: performance API에 공연 자체가 없으면 "podor 미등록 공연"으로 분류.
4. **티켓오픈 정보 파싱 실패**: 정규식으로 추출 시도. 실패하면 "파싱 실패"로 분류하여 메일에 포함.
5. **하나의 상세 페이지에 여러 오픈 차수**: 각 차수별 별도 레코드로 처리.
6. **스크래핑 결과 0건**: 스크래핑 오류 가능성 경고 메일 발송.

## 메일 양식

```
[NOL 티켓 오픈 알림] 2026-03-16

■ 신규 발견 2건 (등록 가능):
1. 은밀하게 위대하게 | 마지막 | 3월 17일 11시 | 4월 11일-4월 26일
2. 팬레터 | 3차 | 3월 19일 14시 | 4월 28일-5월 17일

■ podor 미등록 공연 1건 (공연정보 자체가 없음):
- "뉴뮤지컬 OO" — podor에 공연 등록 필요

■ 시즌 미등록 1건 (공연은 있으나 해당 시즌 없음):
- "홍련" — 전체공연기간 2026.03.01~06.30에 해당하는 시즌 없음

■ 파싱 실패 1건 (티켓오픈 정보 추출 불가):
- "XX뮤지컬" — 오픈 일시 형식 인식 실패

■ 매칭 실패 1건 (공연명 추정 불가):
- "OOOO" (인터파크 원문) — podor 공연 목록에서 유사 공연명 못 찾음

■ 이미 등록됨 8건 (스킵)

■ 어린이 뮤지컬 제외 2건:
- 키즈뮤지컬 OO, 가족뮤지컬 XX
```

## 에러 처리 & 안정성

- **스크래핑 타임아웃**: 각 페이지 30초, 전체 GitHub Actions job 15분 제한
- **재시도**: 페이지 로드 실패 시 최대 2회 재시도 (5초 대기)
- **API 토큰**: 실행 시작 시 1회 발급. 단일 실행이 수 분 내 완료되므로 만료 걱정 없음
- **API 데이터**: 전체 목록을 1회 조회하여 메모리에 캐싱 (API에 필터 파라미터 불확실)
- **메일 발송 실패**: stdout으로 전체 리포트 출력 → GitHub Actions 로그에 보존
- **Rate limiting**: 인터파크 페이지 간 1~2초 딜레이

## 프로젝트 구조

```
Ticketopen/
├── .github/
│   └── workflows/
│       └── daily_scrape.yml    # GitHub Actions (cron: 0 0 * * * UTC = 09:00 KST)
├── main.py                     # 진입점 - 전체 흐름 실행
├── scraper.py                  # 인터파크 스크래핑 (Playwright)
├── podor_api.py                # podor API 연동 (로그인, 조회)
├── matcher.py                  # 공연명/공연장/시즌 매칭 로직
├── parser.py                   # 상세 페이지 텍스트 파싱 (오픈일시, 차수 추출)
├── mailer.py                   # 메일 발송 (SMTP)
├── config.py                   # 설정 (필터 키워드 등, 민감정보는 환경변수)
├── tests/                      # 매칭/파싱 로직 단위 테스트
│   ├── test_matcher.py
│   └── test_parser.py
└── requirements.txt            # playwright, requests
```

## 기술 스택

- **Python 3.11**
- **Playwright**: 인터파크 동적 페이지 스크래핑
- **requests**: podor API 호출
- **smtplib** (stdlib): 메일 발송 (Gmail SMTP)
- **GitHub Actions**: 매일 자동 실행 (cron)

## 환경 변수 (GitHub Secrets)

- `PODOR_USER_ID`: podor 로그인 ID
- `PODOR_PASSWORD`: podor 로그인 비밀번호
- `SMTP_EMAIL`: 발송용 Gmail 주소
- `SMTP_PASSWORD`: Gmail 앱 비밀번호
- `NOTIFY_EMAIL`: 수신 메일 (tingsung93@naver.com)
